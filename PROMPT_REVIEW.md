# 프롬프트 검토 기준

AI가 작성한 코드 요청 프롬프트를 아래 기준으로 검토한다.
이 문서는 C++ 초보 학습용 `LocalAIChat` 프로젝트에서, 단계별 학습 목표를 벗어나지 않는 좋은 코드 요청 프롬프트를 만들기 위한 기준이다.

## 기본 검토 원칙

1. 한 번에 하나의 단계 또는 하나의 기능만 요청하는가?
2. 현재 단계의 목표와 이전 단계의 구현 상태를 분명히 설명하는가?
3. 기존 기능을 유지해야 한다는 조건이 포함되어 있는가?
4. 새로 추가할 기능과 아직 금지할 기능을 구분하는가?
5. C++ 초보자가 이해할 수 있는 수준으로 요구하는가?
6. 구현 파일과 작업 위치를 명확히 제한하는가?
7. 실행 방법과 검증 방법을 요구하는가?
8. 빌드 실패, 연결 실패, 파일 없음 같은 오류 상황을 어떻게 처리할지 요구하는가?
9. 결과 보고 형식을 지정하는가?
10. git commit, git push, remote 설정 여부를 명확히 구분하는가?

## 프로젝트 공통 기준

1. 작업 폴더는 `D:\KM\K82\NA\NA` 안으로 제한한다.
2. Visual Studio 2022 C++ 콘솔 앱 프로젝트 구조를 유지한다.
3. CMake로 전환하지 않는다.
4. 새 `.cpp`/`.h` 파일 분리는 사용자가 요청한 단계 전까지 하지 않는다.
5. 기존 기능을 깨뜨리지 않도록 유지 조건을 명시한다.
6. 초보자용 주석과 설명을 요구한다.
7. 가능한 경우 MSBuild 또는 Visual Studio 기준 빌드 검증을 요구한다.
8. git push, git remote add, GitHub 연결은 명시 요청 전까지 금지한다.
9. 런타임 파일(`chat_history.txt`)과 소스 파일의 추적 여부를 구분한다.

## 단계별 검토 기준

### 0단계: 개발 환경 준비

- Visual Studio 2022 네이티브 C++ 콘솔 앱 프로젝트 생성을 요청하는가?
- `LocalAIChat.sln`, `.vcxproj`, `.filters`, `main.cpp` 구조를 명확히 요구하는가?
- C++17, Debug/Release, Win32/x64 구성을 요구하는가?
- 채팅 기능, 사용자 입력, 반복문, `getResponse`를 아직 금지하는가?
- 최소 실행 확인 메시지만 요구하는가?

### 1단계: 콘솔 기반 기본 채팅

- `getline` 기반 사용자 입력을 요구하는가?
- `while` 반복과 `exit` 종료 조건을 요구하는가?
- `getResponse` 함수 분리를 요구하는가?
- `안녕`, `C++`, `도움` 키워드와 기본 응답을 명확히 요구하는가?
- 자료구조, 파일 저장, 실제 AI 연결을 아직 금지하는가?

### 2단계: 함수 중심 구조 정리

- 기존 1단계 동작 유지를 요구하는가?
- `printWelcomeMessage`, `getUserInput`, `isExitCommand`, `printBotResponse` 같은 함수 분리를 요구하는가?
- `main` 함수가 전체 흐름만 보이도록 단순화하라고 요구하는가?
- 새 기능 추가보다 구조 정리를 목표로 하는가?

### 3단계: 메모리 대화 기록

- `struct Message`와 `std::vector<Message>` 사용을 명확히 허용하는가?
- 사용자 입력과 AI 응답을 실행 중 메모리에 저장하도록 요구하는가?
- `기록` 또는 `history` 입력 시 기록 출력 조건을 요구하는가?
- `exit` 입력은 기록에 저장하지 않는다고 명시하는가?
- 파일 저장은 아직 금지하는가?

### 4단계: 명령어 시스템

- `/`로 시작하는 입력을 명령어로 처리하도록 요구하는가?
- 명령어를 `/help`, `/history`, `/clear`, `/exit`로 제한하는가?
- 명령어 처리를 `handleCommand` 함수로 분리하도록 요구하는가?
- 일반 대화는 기존 `getResponse` 흐름을 유지하도록 요구하는가?
- 정식 명령어 외 확장을 미리 구현하지 않도록 제한하는가?

### 5단계: 간단한 응답 알고리즘

- 기존 명령어 시스템을 유지하도록 요구하는가?
- `getResponse` 또는 규칙 기반 응답 함수 개선을 요구하는가?
- `std::string::find` 기반 키워드 검색을 요구하는가?
- 키워드 우선순위가 `if / else if`로 하나의 답변만 선택된다는 점을 명시하는가?
- 실제 AI 연결은 아직 금지하는가?

### 6단계: 파일 저장과 불러오기

- 파일 입출력을 대화 기록 저장/불러오기 목적으로만 허용하는가?
- `saveHistory`, `loadHistory` 함수를 요구하는가?
- 저장 형식 `Speaker|Text`를 명시하는가?
- 프로그램 시작 시 불러오기, `/exit` 시 저장을 요구하는가?
- `/clear`가 메모리 기록 삭제와 저장 파일 비우기를 함께 수행한다고 명시하는가?
- 파일이 없거나 파일 입출력에 실패해도 프로그램이 종료되지 않도록 요구하는가?

### 7단계: 클래스 기반 구조 리팩토링

- 기능 추가보다 클래스 기반 구조 리팩토링이 목적임을 분명히 하는가?
- `Message`, `ChatBot`, `HistoryManager`, `CommandHandler`, `ChatApp` 역할을 요구하는가?
- `main` 함수가 `ChatApp app; app.run();` 흐름으로 짧게 유지되도록 요구하는가?
- `public`, `private`, 생성자, 멤버 변수, 멤버 함수를 학습할 수 있게 요구하는가?
- 상속, 다형성, 템플릿, 람다식은 아직 금지하는가?

### 8단계: 메모리 개념 학습용 기능

- 기존 채팅 구조를 포인터 기반으로 바꾸지 말라고 명시하는가?
- `/memory` 명령어를 요구하는가?
- 값 전달, 참조 전달, 포인터 기본, 동적 할당, 스마트 포인터, `vector` size/capacity 예제를 요구하는가?
- `new/delete` 예제는 짧고 안전하게 작성하며 반드시 `delete`를 호출하라고 요구하는가?
- 스마트 포인터는 `std::unique_ptr` 하나만 사용하도록 제한하는가?
- `/memory` 실행 내용은 대화 기록에 저장하지 않아도 된다고 명시하는가?

### 9단계: 실제 로컬 AI 모델 연결

- Ollama 로컬 서버만 사용한다고 명시하는가?
- 기본 주소 `http://localhost:11434`, 엔드포인트 `/api/generate`, 모델 `llama3.2`를 명시하는가?
- 모델 이름을 코드 상단 `const string`으로 쉽게 바꿀 수 있게 요구하는가?
- 일반 대화에서 먼저 로컬 AI 요청을 시도하고 실패 시 규칙 기반 응답으로 fallback하도록 요구하는가?
- 외부 라이브러리, OpenAI API, 클라우드 API, API Key, 원격 서버 연결을 금지하는가?
- JSON 요청 생성과 응답의 `response` 문자열 추출 과정을 초보자용 주석으로 설명하게 하는가?
- Ollama가 설치되지 않았거나 서버가 꺼져 있어도 프로그램이 종료되지 않도록 요구하는가?
- `/help`에 로컬 AI 사용 안내를 추가하도록 요구하는가?

## 좋은 프롬프트의 구성

좋은 코드 요청 프롬프트는 보통 아래 순서를 따른다.

1. 현재 프로젝트와 단계 설명
2. 이번 단계의 목표
3. 이전 단계에서 유지할 기능
4. 새로 구현할 기능
5. 수정할 파일과 수정하지 말아야 할 파일
6. 허용되는 C++ 개념
7. 금지되는 C++ 개념 또는 기능
8. 오류 처리 조건
9. 검증 방법
10. git 작업 여부
11. 최종 보고 형식

## 자주 부족한 점

1. 기능을 한 번에 너무 많이 요청한다.
2. 이전 단계 기능 유지 조건이 없다.
3. 금지할 기능을 명확히 적지 않는다.
4. 빌드 검증 또는 실행 검증을 요구하지 않는다.
5. 실패 상황에 대한 fallback 조건이 없다.
6. 초보자용 주석과 설명 요구가 없다.
7. 파일 위치와 작업 범위가 모호하다.
8. 커밋은 요구하면서 push 금지 여부를 쓰지 않는다.
9. 런타임 파일과 소스 파일을 구분하지 않는다.
10. 실제 AI 연결처럼 환경 의존성이 있는 작업에서 설치/실행 실패 상황을 다루지 않는다.

## AI에게 요청할 검토 프롬프트

아래 프롬프트 초안을 위 기준에 따라 검토해줘.

검토할 때는 다음 형식으로 답변해줘.

1. 항목별 평가
2. 부족한 점
3. 위험하거나 범위가 넓은 부분
4. 단계별 학습 흐름에 맞지 않는 부분
5. 수정 방향
6. 개선된 최종 프롬프트

검토 기준:

- 한 번에 하나의 단계 또는 기능만 요청하는가?
- 현재 단계에서 허용되는 C++ 개념만 사용하도록 제한하는가?
- 기존 기능 유지 조건이 명확한가?
- 금지 기능이 명확한가?
- 구현 파일과 작업 위치가 명확한가?
- 초보자용 설명과 주석을 요구하는가?
- 빌드 및 실행 검증을 요구하는가?
- 실패 상황과 fallback 조건을 다루는가?
- git commit, git push, remote 설정 여부가 명확한가?
- LocalAIChat의 단계별 학습 흐름과 맞는가?
