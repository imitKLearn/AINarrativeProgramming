# AGENTS.md

## 프로젝트 성격

이 프로젝트는 C++ 초보 학습용 프로젝트입니다. LocalAIChat을 단계적으로 확장하면서 C++ 문법, 프로그램 구조, 자료구조, 메모리, 알고리즘 기초를 익히는 것이 목적입니다.

## 작업 규칙

1. 사용자가 요청한 단계의 범위만 구현합니다.
2. 다음 단계 기능을 미리 구현하지 않습니다.
3. 8단계까지는 `main.cpp` 하나를 중심으로 유지하고, 여러 `.cpp`/`.h` 파일로 분리하지 않습니다.
4. 8단계에서는 메모리 개념 학습용 기능만 추가합니다.
5. 값 전달, 참조 전달, 포인터 기본, 동적 할당, 스마트 포인터, 객체 생명주기 관찰 기능을 추가합니다.
6. 포인터, 동적 메모리 할당, 스마트 포인터는 8단계 메모리 학습 예제 안에서만 제한적으로 사용합니다.
7. 기존 채팅 프로그램의 핵심 구조를 포인터 기반으로 바꾸지 않습니다.
8. 사용할 클래스는 `Message`, `ChatBot`, `HistoryManager`, `CommandHandler`, `ChatApp`으로 제한합니다.
9. 대화 기록 저장과 불러오기 목적에 한해 `std::ifstream`, `std::ofstream`, `chat_history.txt`를 사용합니다.
10. 5단계의 규칙 기반 응답 알고리즘과 4단계 명령어 시스템을 유지합니다.
11. 명령어는 `/help`, `/history`, `/clear`, `/exit`, `/memory`로 유지합니다.
12. 일반 대화 응답은 `ChatBot`에서 `std::string::find` 기반 키워드 검색으로 처리합니다.
13. 대화 기록은 `Message`와 `std::vector<Message>`를 사용합니다.
14. 사용자가 명시적으로 요청하기 전까지 `map`을 사용하지 않습니다.
15. 실제 AI 모델 연결은 후반 단계까지 구현하지 않습니다.
16. Ollama, OpenAI API, HTTP 요청, 서버 기능은 후반 단계까지 구현하지 않습니다.
17. 상속, 다형성, 템플릿 직접 작성, 람다식, `shared_ptr`, `weak_ptr`를 사용하지 않습니다.
18. 복잡한 포인터 연산, 배열 포인터, 이중 포인터, `malloc/free`를 사용하지 않습니다.
19. GUI로 전환하지 않습니다.
20. CMake 프로젝트로 전환하지 않습니다.
21. 코드 설명은 C++ 초보자 기준으로 작성합니다.
22. Visual Studio 2022와 Windows 환경을 우선 기준으로 합니다.
23. 변경 후에는 가능한 범위에서 빌드 검증을 수행합니다.
24. 빌드 검증이 성공하면 로컬 git commit을 생성할 수 있습니다.
25. git push, git remote add, GitHub 연결은 절대 하지 않습니다.
